# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

**Run locally:**
```bash
pip install -r requirements.txt
cp .env.example .env     # then fill in real values
python src/main.py
```

Required env vars: `OPENROUTER_API_KEY`, `RESEND_API_KEY`, `DIGEST_TO_EMAIL`
Optional: `DIGEST_FROM_EMAIL`, `TAVILY_API_KEY`

Secrets load via `python-dotenv` (`.env` is gitignored). `src/config.py` validates required vars at startup and exits with a clear error if any are missing.

## Architecture

The pipeline runs as a single GitHub Actions workflow (`daily-digest.yml`) on a cron schedule. No server required.

**Data flow:** `main.py` calls `load_config()` then orchestrates three stages:

1. **`sources.py`** — Fetches items from three source types into a common `Item` dataclass:
   - `fetch_rss_items` — iterates `RSS_FEEDS` via `feedparser`.
   - `fetch_twitter_items` — iterates `TWITTER_HANDLES` via Nitter RSS (`{instance}/{handle}/rss`); tries each instance in `NITTER_INSTANCES` until one returns entries. Nitter instances die often; the list is meant to be rotated.
   - `fetch_tavily_items` — optional web search if `TAVILY_API_KEY` is set.

   All three feed into `fetch_all_items()`, which dedupes by SHA1 of normalized URL+title, filters by `RELEVANCE_KEYWORDS`, sorts newest-first, and caps at 40 items.

2. **`summarize.py`** — Sends items to OpenRouter (OpenAI-compatible API) using the free `google/gemma-3-27b-it:free` model. Returns `(html, text)` tuple. HTML is generated by a hand-rolled `_markdown_to_html()` converter (no external markdown lib).

3. **`email_sender.py`** — POSTs to Resend API. Uses `onboarding@resend.dev` as default sender (no domain verification needed for personal use).

## Key Customization Points

- **RSS feeds**: `RSS_FEEDS` in `src/sources.py`
- **Twitter handles**: `TWITTER_HANDLES` in `src/sources.py`
- **Nitter instances** (rotate when dead): `NITTER_INSTANCES` in `src/sources.py`
- **Relevance filter**: `RELEVANCE_KEYWORDS` in `src/sources.py`
- **Summary style/tone**: `SYSTEM_PROMPT` in `src/summarize.py`
- **Model**: `MODEL` constant in `src/summarize.py` — any OpenRouter model ID works (free or paid)
- **Schedule**: `cron:` in `.github/workflows/daily-digest.yml` (UTC)
